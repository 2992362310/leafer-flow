# Leafer-Flow 项目分析与后续规划

本文档基于对当前 `leafer-flow` 项目（Commit Date: 2026-01-14）的分析，对比业界标杆 `Excalidraw`，梳理出的现状、差距以及后续开发路线图。

## 1. 现状分析

### 1.1 技术架构
*   **核心引擎**: [LeaferJS](https://www.leaferjs.com/) (高性能 Canvas 2D 渲染引擎)
*   **框架**: Vue 3 + TypeScript + Vite
*   **状态管理**: Pinia
*   **UI 库**: DaisyUI / TailwindCSS

### 1.2 已实现功能
*   **基础图形**: 矩形 (Rect)、圆形 (Circle)、菱形 (Diamond)、文本 (Text)
*   **连线系统**:
    *   基于 `leafer-connector`
    *   实现了 **Point-to-Point** (坐标连线)
    *   实现了 **Node-to-Node** (节点吸附连线，移动节点连线跟随)
    *   自动吸附逻辑 (中心点吸附)
*   **核心编辑**:
    *   选择/移动工具
    *   撤销/重做 (Undo/Redo) 系统
    *   辅助插件: 标尺 (Ruler)、网格 (Dot Matrix)、吸附对齐 (Snap)
*   **代码结构**:
    *   `src/editor`: 核心逻辑封装，采用插件化设计。
    *   `src/editor/tools`: 工具模式 (State Pattern)，易于扩展新工具。

## 2. 对比 Excalidraw 分析

Excalidraw 的核心优势在于**“手绘风格”**、**“极简交互”**和**“数据自由”**。目前 `leafer-flow` 更偏向于标准的流程图工具基础。

| 维度 | Excalidraw 功能特征 | Leafer-Flow 现状 | 差距与建议 |
| :--- | :--- | :--- | :--- |
| **视觉风格** | 手绘风格 (Rough.js), 不规则线条 | 标准矢量风格, 规则几何体 | **差异化定位**。如需模仿 Excalidraw，需引入手绘渲染；否则保持专业绘图风格。 |
| **属性系统** | 侧边栏实时编辑：颜色、填充、粗细、透明度、图层 | **无** (目前样式在创建时硬编码) | **🔴 极高优先级**。必须实现属性面板，否则只能看不能改。 |
| **绘图工具** | 矩形、圆、菱形、箭头、线段、**自由绘制(铅笔)** | 矩形、圆、菱形、箭头、文本 | 缺少 **线段** 和 **自由画笔**。 |
| **数据能力** | 本地存储、导出 (PNG/SVG/JSON)、协作 | **无** (浏览器刷新即丢失) | **🔴 高优先级**。需实现 JSON 序列化存储与导出图片。 |
| **高级编辑** | 编组 (Group)、锁定、多选对齐、层级调整 | 基础单选/拖拽 | 缺少 **编组**、**层级管理 (Z-index)**。 |
| **交互细节** | 快捷键 (Ctrl+C/V, Del, Arrows)、右键菜单 | 基础点击交互 | 需补充快捷键系统和右键菜单。 |

## 3. 后续开发路线图 (Roadmap)

建议按照以下阶段进行迭代，优先解决“可用性”问题。

### 📅 第一阶段：核心交互闭环 (Make it Editable)
目标：用户画完图形后，可以修改它的样式和属性。

1.  **开发属性面板 (`PropertiesPanel`)**
    *   监听编辑器的 `selection` 事件。
    *   UI 实现：右侧面板，包含颜色选择器、滑动条 (线宽/透明度)。
    *   逻辑实现：选定图形 -> 读取属性 -> 面板回显 -> 修改面板 -> 实时更新图形。
2.  **完善快捷键系统**
    *   `Delete` / `Backspace`: 删除选中元素。
    *   `Ctrl + C` / `Ctrl + V`: 复制粘贴 (Clipboard API 或 内部克隆)。
    *   `Arrow Keys`: 像素级微调移动。

### 💾 第二阶段：数据持久化 (Save & Load)
目标：用户刷新页面不丢失数据，并且可以将成果导出。

1.  **JSON 导入导出**
    *   利用 Leafer 的 `app.tree.toJSON()` 和 `app.tree.add(json)`。
    *   实现 "保存文件 (.json)" 和 "打开文件" 功能。
2.  **自动保存 (Auto-Save)**
    *   监听画布变化 (`app.tree.on('change', ...)`)。
    *   防抖写入 `localStorage`。
3.  **图片导出**
    *   实现 "导出为 PNG" 功能 (`app.export('png')`)。

### 🛠 第三阶段：增强编辑能力 (Advanced Features)
目标：支持更复杂的绘图场景。

1.  **图层管理 (Layering)**
    *   实现 "置于顶层"、"置于底层"、"上移/下移"。
2.  **编组功能 (Grouping)**
    *   实现 `Group` 概念，允许用户将多个元素合并为一个整体进行操作。
3.  **新增工具**
    *   **自由绘制 (Pencil)**: 记录鼠标轨迹生成 `Line` 或 `Path`。
    *   **线段工具**: 简单的直线绘制。
    *   **图片插入**: 支持从本地拖拽图片到画布。

### 🎨 第四阶段：UI/UX 优化
1.  **界面布局重构**
    *   顶部：工具栏 (居中)。
    *   左侧：图层面板/页面管理 (可选)。
    *   右侧：属性面板 (即时响应)。
2.  **帮助与指引**
    *   添加快捷键提示。
    *   欢迎/引导页。
